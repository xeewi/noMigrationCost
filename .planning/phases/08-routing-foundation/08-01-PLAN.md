2. Create a private `deriveView` function:
   - Takes `rawHash: string` (may or may not include leading `#`)
   - Strips leading `#` if present: `const hash = rawHash.startsWith('#') ? rawHash.slice(1) : rawHash;`
   - Returns `'docs'` if `hash.startsWith('/')`, otherwise `'calculator'`
   - Per D-11: base64url alphabet (RFC 4648) never produces `/`, so `startsWith('/')` is a lossless namespace discriminator

3. Export `useHashRoute` function:
   - `const [view, setView] = useState<View>(() => deriveView(window.location.hash));` — lazy initializer prevents flash of wrong view on deep-link (per RESEARCH.md Pitfall 2)
   - `useEffect` that adds `hashchange` event listener calling `setView(deriveView(window.location.hash))`, with cleanup via `removeEventListener` (per RESEARCH.md Pitfall 3)
   - `navigateTo` callback via `useCallback`:
     - If `target === 'docs'`: `window.location.hash = sectionId ? '/docs/' + sectionId : '/docs';`
     - If `target === 'calculator'`: `window.location.hash = '';`
   - Return `{ view, navigateTo }`

Imports needed: `useState`, `useEffect`, `useCallback` from `'react'`.
