## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Table cell text, input labels, descriptions |
| Label | 14px | 600 | 1.4 | Field labels, column headers, section titles |
| Heading | 20px | 600 | 1.2 | Card section headings (e.g. "Team Composition") |
| Display | 32px | 600 | 1.1 | Total standalone cost number in summary card |

Font: Inter. Fallback: ui-sans-serif, system-ui, -apple-system.

Two weights only: regular (400) for body, semibold (600) for label/heading/display. Display role achieves visual separation through size (32px vs 20px), not weight.

## Color

Using shadcn default theme (zinc base, CSS variables). All values reference shadcn semantic tokens.

| Role | Shadcn Token | Usage |
|------|-------------|-------|
| Dominant (60%) | `background` / `card` | Page background, input column surface |
| Secondary (30%) | `muted` / `muted-foreground` | Output column card, table row alternation, inactive tab background |
| Accent (10%) | `primary` | Active time-horizon button, active sizing tab indicator, derived-hours label |
| Destructive | `destructive` | Input field validation error — border ring on invalid fields |

Accent reserved for:
- Active state of the 4 time-horizon preset buttons (1, 3, 5, 10 years)
- Active tab indicator on the Story Points / Direct Hours tab switch
- The derived development hours value (to signal it is a computed output, not an input)

Note: Accent is NOT used for: form field borders, table headers, or the total cost display number. The cost number uses `foreground` at display weight to stand on its own.

## Interaction Contract

### Seniority Grid (TEAM-01 through TEAM-05)

- Fixed 4 rows: Junior, Mid, Senior, Lead
- Each row: seniority label | headcount `<Input type="number" min="0">` | €/h rate `<Input type="number">` with default pre-filled
- Headcount = 0 means that level is excluded from the team average (D-08)
- Team average loaded hourly cost: calculated in real-time below the grid, format `€{value}/h avg` (D-09)
- Rate field shows placeholder default (e.g. "40") — user override replaces it immediately

### Sizing Tab Switch (SIZE-01 through SIZE-05)

- `<Tabs defaultValue="story-points">` with two tabs: "Story Points" and "Direct Hours"
- Story Points tab: SP field + velocity field + sprint duration `<Select>` → derived hours shown below as read-only computed value (D-11)
- Direct Hours tab: hours/days `<Input>` + `<Toggle>` for "hours" / "days" mode (D-12)
- Derived development hours shown below active tab in accent color with label "Estimated development hours:" (D-13)

### Time Horizon Buttons (TIME-01 through TIME-02)

- Four `<Button>` elements in a horizontal group: "1 yr" | "3 yrs" | "5 yrs" | "10 yrs"
- Active button: `variant="default"` (accent/primary fill)
- Inactive buttons: `variant="outline"`
- Default active: "5 yrs" (pre-selected on load)
- No slider — button group only (D-14)

### Cost Output Panel (COST-01 through COST-02)

- Summary `<Card>` at top of output column: displays total standalone cost as display-size number (D-15)
- Format: `€{amount}` with thousand separators (e.g. `€124 800`)
- Below summary card: cost breakdown `<Table>` (D-16)
- Breakdown columns: Category | Hours | Cost (€) | Percentage
- Phase 4 rows: Initial Development, Annual Maintenance (×N years), **Total** (D-17)
- Percentage column uses `<Badge variant="secondary">` with value like "62%"
- All values recalculate as inputs change with no Calculate button (D-18)

### Real-Time Recalculation

- No submit or calculate button anywhere in Phase 4
- All formula outputs update on every input `onChange` event
- Derived hours field and team average update on keystroke

## States Reference

| UI State | Trigger | Behavior |
|----------|---------|----------|
| Zero team | All headcounts = 0 | Output card shows empty state copy; cost displays as "—" |
| Zero feature size | SP = 0 or hours = 0 | Output card shows empty state copy; cost displays as "—" |
| Valid | At least 1 team member and size > 0 | Output card shows calculated total and breakdown table |
| Input error | Non-numeric or negative value | Inline error below field; offending field gets destructive border ring |
| Rate override | User edits a default €/h value | Value immediately reflects in team average; no confirmation |
