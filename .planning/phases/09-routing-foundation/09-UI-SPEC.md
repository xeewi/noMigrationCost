### New UI Elements in AppHeader

Nav group — placed between the title (left) and the action buttons (right):

```tsx
<nav className="flex items-center gap-1" aria-label="Main navigation">
  <a
    href="#"
    onClick={(e) => { e.preventDefault(); onNavigate('calculator'); }}
    className={cn(
      "text-sm font-medium px-2 py-1 rounded transition-colors",
      view === 'calculator'
        ? "text-foreground underline underline-offset-4 decoration-1"
        : "text-muted-foreground hover:text-foreground"
    )}
    aria-current={view === 'calculator' ? 'page' : undefined}
  >
    Calculator
  </a>
  <a
    href="#/docs"
    onClick={(e) => { e.preventDefault(); onNavigate('docs'); }}
    className={cn(
      "text-sm font-medium px-2 py-1 rounded transition-colors",
      view === 'docs'
        ? "text-foreground underline underline-offset-4 decoration-1"
        : "text-muted-foreground hover:text-foreground"
    )}
    aria-current={view === 'docs' ? 'page' : undefined}
  >
    Documentation
  </a>
</nav>
```

Element: `<a>` tags, not `<button>` — correct semantic for navigation links; supports `href` for browser-native behavior and accessibility.

Active indicator: `underline underline-offset-4 decoration-1` (Tailwind) + `text-foreground`. No bold weight change — underline alone is sufficient and matches shadcn's minimal aesthetic.

Inactive state: `text-muted-foreground` with `hover:text-foreground` transition. The `transition-colors` class applies the shadcn default transition duration.

Header layout order (left → right): `[App Title] [Nav Group] [Copy Link] [Reset All (calculator only)]`

## Interaction Contract

### View Switching

| Interaction | Trigger | Result | Animation |
|-------------|---------|--------|-----------|
| Navigate to Docs | Click "Documentation" nav link | `view` → `'docs'`; calculator div gets `hidden` class; docs placeholder renders | None — instant (D-05) |
| Navigate to Calculator | Click "Calculator" nav link | `view` → `'calculator'`; `hidden` removed; docs view unmounts | None — instant (D-05) |
| Browser Back/Forward | `hashchange` event | `view` re-derived from `window.location.hash` | None |
| Deep-link to Docs | Page load with `#/docs` or `#/docs/{id}` hash | `useHashRoute` lazy initializer sets `view = 'docs'` before first render | None — no flash |

### Conditional Header Buttons

| Button | Calculator View | Docs View |
|--------|----------------|-----------|
| Copy Link | Visible — copies calculator state hash URL | Visible — copies `#/docs` or `#/docs/{section-id}` URL |
| Reset All | Visible — triggers confirmation dialog | Hidden (`view !== 'calculator'`) |

### Docs View Placeholder (Phase 9 only)

The docs view renders a minimal placeholder paragraph. No interactive elements. No empty state needed — the view intentionally shows placeholder content until Phase 10.

```tsx
{view === 'docs' && (
  <div className="max-w-[1280px] mx-auto px-6 py-8 pb-16">
    <p className="text-muted-foreground text-sm">Documentation coming soon.</p>
  </div>
)}
```

## Accessibility Notes

- Nav links use `<a>` with `aria-current="page"` on the active link — correct landmark semantics.
- Nav group wrapped in `<nav aria-label="Main navigation">` — distinguishes from any future secondary navs (e.g. docs sidebar).
- Keyboard navigation: `<a>` tags are natively focusable and activatable via Enter/Space without additional ARIA.
- Color contrast: `text-muted-foreground` (`oklch(0.556)`) on white background achieves ≥ 4.5:1 contrast ratio for WCAG AA at 14px.
