**Team Composition section (per UI-SPEC Component Inventory + D-01, D-02, D-03, D-04):**
- Section heading: `<h2>Team Composition</h2>` (UI-SPEC: "Team Composition")
- HTML `<table>` with 4 columns: Seniority (th), Headcount (th), euro/h (th), Loaded Cost (th)
- 4 rows rendered with `<template x-for="(row, index) in team" :key="row.seniority">`
- Seniority column: `<td x-text="row.seniority"></td>` (read-only label)
- Headcount column: `<td><input type="number" x-model.number="team[index].headcount" min="0" max="50" step="1"></td>`
  - Each input MUST have a `<label>` — use `aria-label` attribute since visible label is the column header: `aria-label="Headcount for [seniority]"` via `:aria-label="row.seniority + ' headcount'"`
- Euro/h column: `<td><input type="number" x-model.number="team[index].hourlyRate" min="0" step="0.5"></td>` (TEAM-03: editable, defaults pre-filled per TEAM-02)
  - aria-label pattern: `:aria-label="row.seniority + ' hourly rate'"`
- Loaded Cost column: `<td x-text="(team[index].headcount * team[index].hourlyRate).toFixed(0) + ' euro/h total'"></td>` (computed, read-only)
- CRITICAL: Use `team[index].property` not `row.property` for x-model mutation (Research Code Examples: "row is a copy in Alpine v3")
- Below table: `<p>Team average loaded cost: <strong class="team-average-value" x-text="formatEur(teamAverageRate)"></strong>/h</p>` (D-04, TEAM-05, UI-SPEC accent color on value)
- TEAM-04 (remove members): Setting headcount to 0 is the removal mechanism (UI-SPEC: "Not in Phase 1... removal means setting headcount to 0, not deleting a row")
