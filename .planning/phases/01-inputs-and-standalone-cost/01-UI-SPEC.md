## Page Layout

**Source:** CONTEXT.md D-13, D-14, D-15 (Claude's discretion on arrangement)

Two-column layout at viewport widths ≥ 960px. Single-column stacked on narrower viewports (responsive bonus, not a primary target per REQUIREMENTS.md).

```
┌─────────────────────────────────────────────────────────────┐
│  Header: "Feature Cost Calculator"                          │
├────────────────────────┬────────────────────────────────────┤
│  LEFT COLUMN (inputs)  │  RIGHT COLUMN (output)             │
│                        │                                    │
│  1. Team Composition   │  Summary Card                      │
│     4-row seniority    │    Total Cost (prominent)          │
│     grid               │                                    │
│                        │  Breakdown Table                   │
│  2. Feature Sizing     │    Category / Hours / Cost / %     │
│     Tab: SP | Hours    │                                    │
│                        │                                    │
│  3. Time Horizon       │                                    │
│     4 preset buttons   │                                    │
└────────────────────────┴────────────────────────────────────┘
```

**Why two-column:** Phase 2 will add a chart to the right column below the cost output. Establishing the two-column structure now avoids a layout refactor in Phase 2.

Left column width: 420px fixed. Right column: fills remaining space. Gap between columns: 32px (xl spacing token).

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding within table cells |
| sm | 8px | Spacing between label and input, row gap in team grid |
| md | 16px | Default padding inside cards and form sections |
| lg | 24px | Section padding (between Team / Feature Sizing / Horizon sections) |
| xl | 32px | Column gap between left and right panel |
| 2xl | 48px | Vertical padding above/below the header |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- Time horizon preset buttons and team grid headcount inputs: `min-height: 44px` (touch target accessibility). This 44px value applies only to `min-height` on interactive controls — it is not a layout spacing token and does not appear in margin or padding declarations.

## Typography

**Source:** Pico CSS 2.1.1 default type scale on system-ui. No custom font loaded (CDN-only, presentation context favors system fonts for offline reliability).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Form labels, table body, helper text |
| Label | 14px | 400 (regular) | 1.4 | Column headers, input sub-labels (e.g. "SP/sprint") |
| Heading | 20px | 600 (semibold) | 1.2 | Section headings (Team Composition, Feature Sizing, Time Horizon, Cost Output) |
| Display | 32px | 600 (semibold) | 1.1 | Total cost figure in summary card |

**Two-weight system: 400 (body/label) and 600 (heading/display).** At 32px, semibold provides sufficient visual dominance for the total cost figure without introducing a third weight. No exceptions.

## Color

**Source:** Pico CSS 2.1.1 light theme defaults. D-13: light theme, projector-optimized.

| Role | Value | Pico Variable | Usage |
|------|-------|---------------|-------|
| Dominant (60%) | #ffffff | --pico-background-color | Page background, left column background |
| Secondary (30%) | #f8f9fa | --pico-form-element-background-color (approx) | Right column output panel background, team grid rows, breakdown table alternating rows |
| Accent (10%) | #0172ad | --pico-primary | Reserved-for list below |
| Destructive | #c0392b | (custom override) | Remove team member action only |

**Accent (#0172ad) reserved for:**
1. Active time horizon button (the selected preset among 1/3/5/10 years)
2. Active feature sizing tab indicator (underline or background on SP vs Hours tab)
3. Team average hourly cost value (the computed €/h number — draws attention to the result)

Accent is NOT used on standard form inputs, table borders, or generic hover states.

**Secondary surface (#f8f9fa):** Applied as `background-color` on the right output column and on the `<article>` wrapper for the cost summary card. Achieved via a `<style>` block overriding Pico's `--pico-card-background-color` for the output panel only.

## Component Inventory

Components required in Phase 1, specified for the executor:

### Team Composition Grid

- Rendered as an HTML `<table>` (Pico styles tables natively)
- 4 rows: Junior, Mid, Senior, Lead
- Columns: Seniority (text), Headcount (`<input type="number">`), €/h (`<input type="number">` with default pre-filled), Loaded Cost (computed, read-only text)
- Below the table: "Team average: €X/h" — updates via Alpine.js computed property
- Remove-row behavior: Not in Phase 1 (TEAM-04 is removing members; with fixed 4-slot grid D-01, removal means setting headcount to 0, not deleting a row)

### Feature Sizing Tabs

- Two tab buttons: "Story Points" and "Direct Hours"
- Active tab: accent (#0172ad) bottom border, 2px, font-weight 600
- Inactive tab: no border, font-weight 400, color #6c757d
- Tab panel for Story Points: SP input + velocity input + sprint duration `<select>` (1 wk / 2 wks / 3 wks / 4 wks)
- Tab panel for Direct Hours: hours input + days input with "× 7h/day" label toggle
- Below tabs: "Estimated development hours: X h" — updates reactively

### Time Horizon Buttons

- Four `<button>` elements in a `<div role="group">`: 1 yr, 3 yrs, 5 yrs, 10 yrs
- Active state: `background-color: #0172ad; color: #fff`
- Inactive state: `background-color: #f8f9fa; color: #373c44; border: 1px solid #dee2e6`
- Height: `min-height: 44px` (touch target — not a spacing token)
- Rendered as a button group (inline, no gaps between buttons — connected borders)

### Cost Summary Card

- `<article>` element (Pico gives it card styling)
- Top: section heading "Standalone Feature Cost" (20px, weight 600)
- Center: total cost figure — large display number (32px, weight 600), formatted as "€XX,XXX"
- Below: secondary label "over {N} year(s)" (14px, weight 400, color #6c757d)

### Cost Breakdown Table

- HTML `<table>` with `<thead>` and `<tbody>`
- Columns: Category, Hours, Cost (€), Share (%)
- Rows: Initial Development, Maintenance, Coordination, Bug Fixing, Sync Overhead
- Footer row (bold): Totals
- No sortable columns in Phase 1 (static, no `class="sortable"`)

## Interaction Contract

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Change headcount | `input` event on headcount field | Team average €/h recalculates immediately |
| Override €/h | `input` event on €/h field | Team average recalculates immediately; overridden field loses default styling |
| Switch feature sizing tab | Click "Story Points" or "Direct Hours" button | Opposite panel hides (`x-show`), active tab indicator updates, derived hours recalculates |
| Change SP / velocity / sprint | `input`/`change` event | Derived hours display updates immediately |
| Change direct hours/days | `input` event | Derived hours display updates (days × 7 if days mode) |
| Select time horizon | Click preset button | Active button highlights (accent), all cost output recalculates |
| All input changes | Any `input`/`change` event | Summary card total and breakdown table update without page reload, no button press required |

**No "Calculate" button exists.** All outputs are Alpine.js computed values — they are always current. (Source: D-12)

## States to Implement

| State | Condition | UI Behavior |
|-------|-----------|-------------|
| Zero state | All headcounts = 0 OR estimated hours = 0 | Output panel shows empty state copy instead of cost figures |
| Partial state | Some headcounts set, hours not set (or vice versa) | Output panel shows empty state copy — both team and size needed |
| Active state | All required inputs non-zero | Summary card shows total, breakdown table shows all rows populated |
| Override state | User has typed into an €/h field | Field value is no longer the research default; no visual badge in Phase 1 (Phase 2 concern) |
| Active tab | SP or Direct Hours tab selected | Tab indicator (accent underline), panel visible |
| Active horizon | One of 1/3/5/10 selected | Button filled with accent color |

## Accessibility Notes

- All `<input>` elements must have an associated `<label>` (explicit `for` / `id` pairing or wrapping)
- Tab panels use `role="tablist"`, `role="tab"`, `role="tabpanel"`, `aria-selected`, `aria-controls` attributes
- Time horizon button group: `role="group"` with `aria-label="Time horizon"`, active button gets `aria-pressed="true"`
- Cost breakdown table: `<caption>` element with "Cost breakdown by category"
- Color contrast: #0172ad on #ffffff = 4.7:1 (passes WCAG AA for normal text at 16px and large text at 14px+)
- Color contrast: #373c44 on #ffffff = 9.7:1 (Pico default text — passes AAA)
