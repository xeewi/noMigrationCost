## Spacing Scale

Declared values (all multiples of 4). Consistent with Phase 4 established patterns.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, info icon padding, inline badge padding |
| sm | 8px | Compact element spacing, slider-to-input gap, card inner row gaps |
| md | 16px | Default element spacing, card section padding, form field stacking |
| lg | 24px | Section padding, gap between summary cards and chart |
| xl | 32px | Layout column gap (existing `gap-8` = 32px) |
| 2xl | 48px | Major section breaks between tabs |
| 3xl | 64px | Page-level vertical padding (existing `py-8` = 32px top/bottom) |

Exceptions:
- Chart height: fixed 320px (not a spacing token — dimension constraint per D-17)
- Touch targets for slider thumb and info icon: minimum 44px hit area (accessibility exception)

## Typography

Inherits Geist Variable from Phase 4 base. No new font families introduced.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Parameter labels, table cells, tooltip copy, citation text |
| Label | 12px | 500 | 1.4 | Category row labels in breakdown table, badge text, axis tick labels |
| Heading | 20px | 600 | 1.2 | Tab labels, summary card headings ("Shared Path", "Duplicated Path") |
| Display | 28px | 700 | 1.1 | Summary card total cost values (€42K format) |

Note: 14px body and 12px label are the two working sizes for dense data tables. Display size (28px) is reserved strictly for the two path total values — it is the visual anchor of the comparison tab.

## Color

Base palette uses oklch neutral tokens from src/index.css. Semantic chart colors are custom additions for this phase.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` oklch(1 0 0) / white | Page background, chart background, card backgrounds |
| Secondary (30%) | `--card` / `--muted` oklch(0.97 0 0) | Cards (summary cards, breakdown table rows), input column backgrounds, advanced parameters panel |
| Accent (10%) | `--primary` oklch(0.205 0 0) / near-black | Active tab indicator, "Reset to defaults" button, primary interactive states |
| Chart: Shared Path | `#16a34a` (green-600) | Shared path area fill, shared path line stroke, legend swatch, summary card left border |
| Chart: Duplicated Path | `#dc2626` (red-600) | Duplicated path area fill, duplicated path line stroke, legend swatch, summary card left border |
| Break-even success | `--color-success` mapped to green-100 bg / green-700 text | Break-even callout when break-even exists (D-07 success state) |
| Break-even warning | amber-50 bg / amber-700 text (Tailwind `amber-50`/`amber-700`) | Break-even callout when no break-even exists (D-07 warning state) |

Accent reserved for: active tab indicator underline, "Reset to defaults" button fill, "Modified" badge outline stroke, slider range fill track.

Chart fill opacity: 20% alpha on area fills (both paths) to allow overlap visibility. Full opacity on strokes.

Break-even reference line: `--muted-foreground` oklch(0.556 0 0) as dashed stroke, 1.5px width.

## Interaction Contracts

### Tabbed Output (D-01, D-02, D-03)

- Default active tab on page load: "Comparison"
- Tab labels: "Comparison" (left) | "Standalone" (right)
- Switching to "Standalone" renders the existing CostOutput component unchanged
- Tab state: local React `useState` inside the output column wrapper — not lifted to App.tsx

### Summary Cards (D-06)

- Two cards side-by-side with equal width: "Shared Path" and "Duplicated Path"
- Each card: colored left border (green / red), path name heading (20px/600), total cost in display size (28px/700)
- Delta/savings: third element below both cards — label "You save" + value in green when shared < duplicated; label "Shared costs more" + value in amber when shared > duplicated
- Standalone total badge (D-04): small `Badge` variant="outline" inside the Comparison tab header area showing "Standalone: €X" — sourced from existing standalone calculation

### Break-Even Callout (D-07)

- Positioned between summary cards and chart
- Two visual states:
  - Break-even exists: `bg-green-50 border border-green-200 text-green-800` rounded card. Copy: "Break-even at [N] months — you save €[Y] over [Z] years"
  - No break-even: `bg-amber-50 border border-amber-200 text-amber-800` rounded card. Copy: "No break-even within [Z] years — shared code costs more in this scenario. Try increasing the number of consuming teams or extending the time horizon."

### Chart (D-10 through D-17)

- Recharts `AreaChart` wrapped in `ResponsiveContainer` (width="100%", height=320)
- Two `Area` components: shared (green-600 stroke, green-600/20 fill) and duplicated (red-600 stroke, red-600/20 fill)
- `XAxis`: dataKey="month", label "Month", tick every 12 months for horizons >= 3 years, every 6 months for 1-year horizon
- `YAxis`: tickFormatter abbreviates to €0K / €42K / €1.2M format (K for thousands, M for millions)
- `Tooltip`: custom content showing "Month [N]" header, Shared: €X, Duplicated: €Y, Difference: €Z (green text when shared < duplicated, amber when not)
- `ReferenceLine`: vertical dashed line at break-even month using `--muted-foreground` color, labeled "Break-even" with small text above the line. Omitted entirely when no break-even.
- Inline legend: `Legend` positioned inside chart area (verticalAlign="top", align="right"), color swatches match path colors
- Chart title: none — summary cards provide the narrative context

### Advanced Parameters (D-18 through D-23)

- `Collapsible` section below the ConsumingTeams card in the left column
- Header row: `CollapsibleTrigger` with ChevronDown/Up icon + "Advanced Parameters" label + conditional `Badge` variant="outline" showing "Modified" when any value differs from ENGINE_DEFAULTS
- Collapsed by default; animates open/close
- Each parameter row: label (14px/400) + `Info` icon (16px, lucide) | `Slider` | `Input` (number, width ~72px)
  - Label and icon left-aligned, slider fills remaining width, input right-aligned
  - Info icon click opens `Popover` with citation text (research section reference, default value, valid range)
  - Slider and input are bound — changing either updates the shared state value
  - Input accepts free-form typing; clamps to valid range on blur
- "Reset to defaults" `Button` variant="outline" size="sm" at the bottom of the expanded section. Copy: "Reset to defaults"
- Seven parameters in order:
  1. Generalization factor — range 1.0–2.0, default 1.3, step 0.05 — citation: research doc §3.2
  2. Porting factor — range 0.3–1.0, default 0.65, step 0.05 — citation: research doc §3.3
  3. Divergence rate — range 0.05–0.50, default 0.20, step 0.01 — citation: research doc §7.5
  4. Maintenance rate (shared) — range 0.10–0.30, default 0.18, step 0.01 — citation: research doc §7.1
  5. Maintenance rate (duplicated) — range 0.15–0.40, default 0.22, step 0.01 — citation: research doc §7.2
  6. Bug duplication factor — range 1.0–3.0, default 2.0, step 0.1 — citation: research doc §7.4
  7. (In main inputs, not here) Consuming teams — see ConsumingTeams card below

### Consuming Teams Card (D-24)

- Separate small Card in the left column, positioned between TimeHorizon and Advanced Parameters
- Label: "Number of consuming teams"
- `Input` type="number" with min=2, max=10, step=1, default=2
- Helper text (12px/400, muted-foreground): "Teams sharing the same codebase"
- Not inside the Advanced Parameters collapsible

## Empty and Error States

### Comparison tab empty state

Triggered when: `teamAvgRate === 0` OR `devHours === 0` (same conditions as standalone empty state).

Display: centered card with muted icon (BarChart2 from lucide, 48px, muted-foreground), heading (20px/600), body text (14px/400, muted-foreground). No chart rendered.

### No break-even state

Not an error — expected outcome. Handled by D-07 warning callout. Chart still renders both curves. ReferenceLine omitted.

### Chart render error

Not expected given pure-function engine. If `calcSharedCost` or `calcDuplicatedCost` returns empty array: fall back to empty state with copy "Cost data unavailable. Check your inputs."
