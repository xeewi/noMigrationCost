  <action>
    Three targeted fixes — no behavioral changes, only type alignment:

    **Fix 1 — src/App.tsx line 51:**
    Change:
    ```typescript
    const [nbConsumingCodebases, setNbConsumingCodebases] = useState(
      ENGINE_DEFAULTS.nbConsumingCodebases,
    );
    ```
    To:
    ```typescript
    const [nbConsumingCodebases, setNbConsumingCodebases] = useState<number>(
      ENGINE_DEFAULTS.nbConsumingCodebases,
    );
    ```
    Why: ENGINE_DEFAULTS is `as const`, so `ENGINE_DEFAULTS.nbConsumingCodebases` has literal type `2`. Without explicit `<number>`, TypeScript infers `useState<2>` making `setNbConsumingCodebases` typed as `Dispatch<SetStateAction<2>>`, which is incompatible with `ConsumingTeams`'s `onChange: (value: number) => void`.

    **Fix 2 — src/components/AdvancedParameters.tsx line 123 and 165:**
    Change the `handleSliderChange` function at line 123 from:
    ```typescript
    function handleSliderChange(newValue: number) {
      onValueChange(newValue);
    }
    ```
    To remove the named function entirely, and at line 164-165 change the Slider from:
    ```tsx
    <Slider
      ...
      onValueChange={handleSliderChange}
    ```
    To:
    ```tsx
    <Slider
      ...
      onValueChange={(v) => onValueChange(Array.isArray(v) ? v[0] : v)}
    ```
    Why: base-ui SliderPrimitive.Root.Props defines `onValueChange` as `(value: number | readonly number[], eventDetails: SliderRootChangeEventDetails) => void`. The inline handler accepts the union type and extracts the scalar.

    **Fix 3 — src/components/AdvancedParameters.tsx line 199:**
    Change:
    ```tsx
    <CollapsibleTrigger asChild>
      <button
        type="button"
        className="flex w-full items-center gap-2 rounded-lg border border-border bg-card px-4 py-3 text-left hover:bg-muted/50 focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring"
      >
        ...children...
      </button>
    </CollapsibleTrigger>
    ```
    To:
    ```tsx
    <CollapsibleTrigger
      className="flex w-full items-center gap-2 rounded-lg border border-border bg-card px-4 py-3 text-left hover:bg-muted/50 focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring"
    >
      ...children (ChevronDown, span, Badge — keep all identical)...
    </CollapsibleTrigger>
    ```
    Why: base-ui CollapsiblePrimitive.Trigger renders as a `<button>` natively. It does not have an `asChild` prop (that is a Radix pattern). Remove the `asChild` prop and the inner `<button>` wrapper, moving className and children directly onto CollapsibleTrigger.
